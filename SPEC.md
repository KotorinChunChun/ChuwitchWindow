# ウィンドウ配置入れ替えツール 要求仕様書

## 1. 概要
複数のディスプレイ環境において、ショートカットキー操作によって、現在開いているウィンドウ群を別のディスプレイへ一斉移動・交換させるWindows向けの常駐デスクトップアプリケーション。

- **英名**: ChuwitchWindow（実行ファイル名等に使用）
- **和名**: ウィンドウ入れ替えツール（タイトルバー、ショートカット名等に使用）

## 2. 開発環境・実行環境
- **対象OS**: Windows 10 / 11
- **開発言語**: Rust (Tauri 2.0 + windows-rs を推奨)
- **設定保存**: `%APPDATA%\com.kotorichun\chuwitchwindow\config.json`（レジストリは使用しない）
- **ログ出力**: `%APPDATA%\com.kotorichun\chuwitchwindow\logs\`（エラーログ等を一時保存）
- **配布形式**: NSISによるインストーラー版（`setup.exe`）および単体 `.exe` 形式
- **多重起動防止**: ミューテックス（Mutex）を利用。既に起動している場合は既存プロセスの設定画面をアクティブにする。

## 3. 主要機能

### 3-1. 機能1：ディスプレイの巡回移動（シフト）
- **動作**: 全ての対象ウィンドウを次のディスプレイに移動させる。※モニターグループ機能と連携
  - 増加方向: モニタ 1 → 2、2 → 3、3 → 1
  - 減少方向: モニタ 1 → 3、3 → 2、2 → 1
- **ショートカットキー(デフォルト)**: 
  - 増加方向 : `Win + Ctrl + Alt + 右矢印`  
  - 減少方向 : `Win + Ctrl + Alt + 左矢印`
  - 設定画面で変更可能。

### 3-2. 機能2：指定モニタとの入れ替え
- **動作**: ユーザーが定義した「プライマリモニター」と、入力したキー番号の「ターゲットモニタ」上のウィンドウを入れ替える。
- **ショートカットキー(デフォルト)**: 
  - 入れ替え発動: `Win + Ctrl + Alt + 2 ～ 9` （数字はターゲットのモニタ番号）
  - アンドゥ（元に戻す）: `Win + Ctrl + Alt + Z` （直前の入れ替えをもとに戻す）
  - 設定画面で変更可能。

### 3-3. 機能3：モニターグループ設定
- **動作**: 事前に設定画面で定義した「同じグループ色のモニタ」のウィンドウのみを一斉に入れ替える。（異なるグループのモニタへは移動しない）設定画面の「モニター構成」タブよりグラフィカルに変更可能。

### 3-4. 機能4：ディスプレイのホットプラグ検知
- **動作**: ディスプレイの抜き差し（構成変更）を自動検知する（`WM_DISPLAYCHANGE`メッセージのフック）。
- **自動回収**: ディスプレイが切断された際、切断された画面に取り残されたウィンドウを、プライマリモニターへ自動的に移動・回収する。

## 4. ウィンドウ制御に関する詳細要件
- **解像度・DPI混在環境の補正**: 
  - 移動先のモニタの解像度（幅・高さ）と元のモニタの比率を算出し、ウィンドウのサイズおよび相対座標をスケーリングして配置する。
  - DPI（スケーリング設定）が異なるモニタ間でも、見かけ上のサイズが維持されるよう計算を最適化する。
- **スナップレイアウトの維持・再構築**:
  - 移動元のウィンドウが画面の正確な半分（50%）や四半部（25%）を占めている（Windowsスナップ機能による配置）場合、単純な比率計算ではなく、移動先の画面領域に対しても正確な50%・25%の比率でスナップし直す補正ロジックを適用する。
- **Zオーダーの完全復元**: 
  - 移動処理を行う前に各ウィンドウのZオーダー（重なり順）を取得・保持し、移動先のモニタでも元の重なり順を完全に復元する。
- **モニターの個体識別**: 
  - モニターの識別にはシステムIDに加え、デバイス名やシリアル番号等の個体情報を組み合わせて利用し、構成変更時にも設定を一貫させる。
- **サイズ変更不可のウィンドウ**: OS側でサイズ変更が不許可になっているウィンドウについては、比率変形を行わず移動先の想定座標中央等に配置する。
- **最大化・最小化**: 対象ウィンドウが最大化・最小化されている場合、移動先でもその状態を維持する。
- **管理者権限ウィンドウへの対応**: 
  - タスクマネージャー等の管理者権限で実行されているウィンドウを操作するため、本アプリ自身の実行権限を管理する。
  - 起動時に管理者権限への昇格を試行するか、設定で切り替え可能にする。
- **自動フィルタリング（バグ防止）**: 
  - **仮想デスクトップ対応**: Windowsの仮想デスクトップ機能において、現在アクティブでない（見えていない）デスクトップに存在するウィンドウは移動対象から外す。
  - **不可視ウィンドウの除外**: バックグラウンド等で動作しているシステムウィンドウや、不可視のウィンドウを誤って移動させないよう自動除外する。
  - **フルスクリーンアプリの除外（設定可）**: 
    - 現在フルスクリーン状態で表示されているアプリ（ゲームや動画プレイヤー等）を移動対象から除外する。
    - 設定画面でこの機能のON/OFFが可能。

## 5. UIおよび常駐機能

### 5-1. システムトレイ常駐
- 終了時や最小化時はタスクトレイに格納される。
- 設定画面を開いたときは最大化した状態で表示する。
- **トレイアイコンの動作**:
  - ダブルクリック: 設定画面を再表示。
  - 右クリックメニュー: 「設定」「再起動」「終了」を表示。

### 5-2. 設定画面（GUI）
- **タブ構成**: 
  - **モニター構成**: モニターの物理配置をグラフィカルに表示。クリックでグループ色を変更。
  - **設定・常駐**: ショートカットキーの変更、スタートアップ登録、管理者権限での再起動などを設定。
  - **アプリについて**: バージョン情報、GitHubリンク、設定のリセット。
- **ショートカットキー設定**: ユーザーが任意のキーコンビネーションに変更可能。競合検知機能付き。
- **スタートアップ設定**: Windowsログイン時の自動起動切り替え。（管理者権限で実行中の場合は管理者権限でのスタートアップ登録となる）
- **管理者権限の動的昇格**: 既定では通常権限で動作。設定画面より「管理者として再起動」が可能。
- **Undo（元に戻す）**: `Win + Ctrl + Alt + Z` により直前の入れ替えをもとに戻す。
- **設定のリセット**: 設定ファイルの内容をリセットし、初期状態に戻す機能。
- **クリーンアップ**: 「設定を完全に削除して終了」ボタンにより、AppData内の設定およびログファイルを消去して終了する。

### 5-3. 視覚的通知機能 (OSD)
- 切り替え直後に影響を受けたモニタの中央へ移動先（例：`1 → 2`）を数秒間表示。
- 設定でON/OFFおよび表示時間を調整可能。
