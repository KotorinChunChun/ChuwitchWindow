## 1. 概要

v0.1 の基本機能（ウィンドウのモニター間一斉移動、入れ替え、グループ化等）を拡張し、操作対象となるウィンドウをより細かく制御するための機能や、他のツールと連携するためのCLI機能などを追加する。

## 2. 追加機能の詳細

### 2-1. ウィンドウの一時的なピン留め機能（ショートカットキー）

現在フォーカスが当たっている特定のウィンドウを、ショートカットキーで一時的に「現在のモニターから動かさない」ようにロック・ピン留めする機能。

- **動作（操作方法）**:
  - 対象のウィンドウをアクティブにした状態で、特定のショートカットキー（デフォルト: `Win + Ctrl + Alt + P`）を入力することで、そのウィンドウのピン留め状態をトグル（ON / OFF）する。
- **対象の識別と保持期間**:
  - 内部的にはウィンドウハンドル（`HWND`）を使用して個別にピン留め状態を管理する。
  - ピン留め状態は `config.json` には保存せず、本ツールを終了するか該当プロセスが終了するまでの一時的な状態とする（永続化したい場合は `2-2` の除外設定を利用する）。
- **フィードバック・通知**:
  - レスポンス低下や操作の妨げを防ぐため、ピン留めのトグルに伴う音や通知ポップアップなどのフィードバック機能は実装しない。

### 2-2. 特定アプリケーションの永続的な除外設定（ルールベース）

設定画面にあらかじめ登録しておいた条件に合致するウィンドウを、本ツールの移動対象から常に除外する機能。

- **動作・条件指定**:
  - 以下の属性を組み合わせて、除外対象とするウィンドウを判定する。
    - **プロセス名**: `.exe` ファイル名
    - **クラス名**: ウィンドウのクラス名
    - **タイトル名**: ウィンドウのタイトル（部分一致等のクエリ想定）
    - **スタイル種別**: ウィンドウのスタイル属性等
  - これらの条件を `AND` や `OR` で組み合わせて、柔軟な除外ルール（クエリ）を作成可能にする。
  - 具体例としてはEntraIDのグループ条件式のような形式をUIを想定する。
- **GUIでの設定機能**:
  - 設定画面に「除外リスト」タブを新設する。
  - **ウィンドウ選択機能**: 現在開いているウィンドウの属性を自動取得し、プルダウンリストから選択して除外ルールを追加できるUIを提供する。
  - **ピン留め状態の提案機能**: `2-1` で一時的にピン留めされているウィンドウが存在する場合、ルール登録一覧において「このピン留めウィンドウを永続的な除外ルールとして登録する」ための提案材料（プルダウンの初期選択状態やサジェスト等）として優先提示する。
  - **リロード機能**: プルダウン内のウィンドウ一覧を最新状態に手動更新するための「リロードボタン」を配置する。
  - **クエリ編集機能**: 追加された除外条件全体が **JSONフォーマット** で一覧表示される、複数行の編集可能なテキストボックスを用意する。ユーザーのタイピングによる直接修正や、コピー＆ペーストによる一括書き換えを可能にすることで、確実な設定変更を実現する。
- **設定の保存**:
  - 設定した除外ルールは JSON フォーマットで `config.json` に保存し、次回起動時も永続的に適用する。

### 2-3. 特定モニターの「一斉退避（エスケープ）」と「一極集中（ギャザー）」

単なるモニター間の「入れ替え」ではなく、一時的に特定の画面を空にしたり、逆に一箇所に集めたりする機能。

- **ターゲットモニターの決定方法**:
  - ショートカットキーを入力した際、「現在アクティブなウィンドウが存在するモニター（ウィンドウが複数モニターにまたがっている場合は、表示面積が最も大きいモニター）」を操作対象モニター（ターゲットモニター）として自動定義する。
- **一斉退避（エスケープ）**:
  - **ユースケース**: 登壇やオンラインミーティングで「モニター2」を画面共有する直前に、モニター2のウィンドウをアクティブにした状態でショートカットキーを押し、そこにあるすべてのウィンドウを他のモニターへ逃がし、クリーンなデスクトップを見せる。
  - **実装案**: ショートカットキー（デフォルト: `Win + Ctrl + Alt + E` _Escapeを意図_）を入力すると、ターゲットモニターに存在する全ウィンドウを他のモニター（プライマリモニター等）へ移動させるロジックを追加する。
- **一極集中（ギャザー）**:
  - **ユースケース**: 作業の区切りがついたとき、散らばった全てのウィンドウをとりあえずアクティブなモニターに集約して整理し直す。
  - **実装案**: ショートカットキー（デフォルト: `Win + Ctrl + Alt + G` _Gatherを意図_）を入力すると、全てのモニターからターゲットモニターへウィンドウを一斉移動させるロジックを追加する。

### 2-4. 外部デバイス・スクリプト連携用のコマンドライン実行引数サポート

キーボードショートカットだけでなく、外部から本ツールの機能を呼び出せるようにする連携機能。

- **ユースケース**:
  - Stream Deckなどの物理マクロパッドのボタンに「モニター1と2の入れ替え」を割り当てる。
  - 自作のバッチファイルやPowerShellスクリプトから、作業開始時にウィンドウ配置をシフトさせる処理を自動化する。
- **実装案**:
  - 実行ファイル（`ChuwitchWindow.exe`）に `--rotate-cw`（増加方向へシフト）、`--rotate-ccw`（減少方向へシフト）、`--undo`（元に戻す）、`--escape`（エスケープ）、`--gather`（ギャザー）、`--swap=N`（モニターNと入れ替え）、`--pin-toggle`（ピン留めトグル） といった起動引数を持たせて実行できるようにする。
  - **常駐済みの挙動**: すでにタスクトレイに常駐しているプロセスが存在する場合は、多重起動防止（Mutex）を活用し、名前付きパイプ（Named Pipe）等を通じて常駐しているメインプロセスにコマンドだけを送信し、自身は即座に終了する設計とする。
  - **未常駐時の挙動**: ツール自体が起動していない状態でCLIから引数付きで呼ばれた場合は、指定された処理（例：シフト移動）を1回だけ実行し、**常駐せずに即座に終了する（完全なワンショットコマンドとして動作する）**。

### 2-5. 最小化されたウィンドウの処理オプション

最小化されたウィンドウに対する移動の振る舞いを選択できるオプション機能。

- **動作**:
  - 既存仕様の「移動先でも最小化された状態を維持する」設定に加え、「最小化されているウィンドウは移動対象から完全に除外する（元のモニターに残す）」という設定オプションを新設する。**（既定値: ON）**
- **目的・メリット**:
  - バックグラウンドで大量に開かれ、最小化されているエクスプローラーやブラウザが、一斉入れ替えのたびに内部的に座標計算・移動処理されるのを防ぐ。
  - 移動対象となるウィンドウの絶対数を減らすことで、Win32 APIの呼び出し回数を抑え、一斉移動時のレスポンス（パフォーマンス）を向上させる。

### 2-6. ウィンドウの自動整列機能

特定のモニター内にあるウィンドウを自動で整理・整列する機能。

- **動作原理**:
  - ピン留め機能等で固定されたウィンドウ、除外リストに登録された例外ウィンドウ、および **サイズ変更不可なウィンドウ（リサイズ枠を持たないもの等）** を「障害物（固定領域）」として扱い、その位置・サイズをそのまま保持する。
  - モニターのワークエリアから、それらの固定領域を差し引いた「残りの空き領域」を計算する。
  - 計算された空き領域に対し、そのモニターに存在する「残りの移動対象ウィンドウ」を均等に配置計算してリサイズ・移動を行う。
- **UIと操作（視認性重視の選択画面）**:
  - 特定のショートカットキー（デフォルト: `Win + Ctrl + Alt + A`）を入力すると、ウィンドウの整列方法を選択する画面が出現する。
  - **対象モニターの決定と記憶**: ポップアップを表示した瞬間のフォアグラウンドウィンドウを元に対象モニターを決定し、記憶しておく。これにより、ポップアップ操作中にフォーカスが奪われても対象モニターが誤判定されないようにする。
  - **視認性とデザイン重視のUI**: 当初はWebViewを避ける想定だったが、リッチなデザイン（透過や背景ブラー）と視認性を優先し、TauriのWebViewを利用したボーダーレスの透過ウィンドウとして実装。画面横幅を広く取り、視認性の高い巨大な正方形ボタンを横並びに配置する。
  - **操作の受付**: マウスによるボタンクリックでの決定に加えて、キーボードの `1`, `2`, `3`, `4` キーを入力することで該当の整列処理が即座に実行されUIが閉じる、一発選択の操作をサポートする。
  - **最小化されたウィンドウの処理**: 最小化されているウィンドウは整列対象から除外する。
  - 除外登録されたウィンドウは移動や変形は行わないが、そこは使用済みとして他のウィンドウを空いた場所に配置する。
  - フォーカスから外れたらウィンドウは閉じて中止とする
  - 「重ねて表示」では、Zオーダーの低いウィンドウ順で左上の背面となるように配置する。
  - 配置方法は以下の4択とし、直感的なアイコンを用いて視覚的にレイアウトを図示する。

    ```text
    [1] 並べて表示
    ┌┬┐
    ├┼┤
    └┴┘

    [2] 上下に並べて表示
    ┌──┐
    ├──┤
    └──┘

    [3] 左右に並べて表示
    ┌┬┬┐
    ││││
    └┴┴┘

    [4] 重ねて表示
     ┌─┐
     └┌─┐
      └─┘
    ```

## 3. 設定画面（GUI）の追加項目

設定画面（GUI全体および各タブ）に以下の機能を追加する。

- **キーボード操作によるタブ移動**:
  - `Ctrl + Tab` および `Ctrl + Shift + Tab` を用いて、ウィンドウ内の設定タブを切り替える操作をサポートする。
- **設定ファイルのインポート・エクスポート**:
  - 現在の設定ファイル（`config.json`）を外部に保存したり、保存したファイルから復元したりできるボタン。
- **CLI用Path環境変数追加**:
  - 環境変数 `Path` に本ツールのディレクトリを追加し、コマンドプロンプトやPowerShell等から `ChuwitchWindow` だけで呼び出せるように登録・解除するボタン。
- **実行ログの表示と管理**（「アプリについて」タブ内）:
  - ツールの稼働状況やエラー等の内部動作ログをテキスト表示し、「更新」および「全消去」ができるUIを提供する。ログの内容は画面に合わせて適切にフォーマットする。

## 4. 既存機能に関する考慮・連携事項

- **フルスクリーン除外との関係**:
  - 既存の「フルスクリーンウィンドウの除外」機能は、上記の「除外設定（クエリ）」や「ピン留め」設定に先立って評価され、最優先の除外ルールとして引き続き機能する。
- **Zオーダー（重なり順）への影響**:
  - 除外またはピン留めされたウィンドウは、モニター間を移動せずに元の位置に留まる。
  - 他のモニターから移動してきたウィンドウ群を配置する際は、これらの「動かなかったウィンドウ」との重なり合いが不自然にならないよう、元のZオーダーの相対関係を可能な限り維持して配置処理を行う。
