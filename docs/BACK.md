## 開発の背景

作者は日頃から3台～6台のモニターを使用していますが、言うまでもなく自分の真正面にあるメインモニターが最も見やすいです。メインモニタには注視するアプリのウィンドウを置いて、サブモニターには補助的な情報を表示するウィンドウを配置します。

ところが、普段はサブモニタに置いているアプリでも、時と場合によってはメインモニターに移動させたくなるときがあります。このとき、メインモニターにあったアプリの前に持ってくると、せっかく重ならないように整列していたウィンドウが重なり、サブモニタには空白ができ、秩序が崩れてしまいます。

また、一般的にモニタに表示されているウィンドウは1つだけではなく、2つ～4つのアプリが同時に見えるように配置されていることが多いです。そのため、いっそのことモニタの中に映っているウィンドウすべてを入れ替えてしまいたいと以前より考えていました。

もちろん、解決策を以前からずっと模索してきました。しかし、いずれの機能・ツールも、私の期待するような動作を提供していませんでした。

## 類似ツール

Windowsにはアクティブなウィンドウの位置を他のモニタへ移動させるショートカットキー `Win + Shift + 左右矢印` がありますが、複数のウィンドウを同時に移動させる手段はありません。もちろん、移動先にあったウィンドウと入れ替えるなんて方法も存在しません。

Windowsの仮想デスクトップ機能 `Win + Ctrl + D` と `Win + Ctrl + 左右矢印` は、少ないモニタで多くのアプリを入れ替えて表示できるという便利な機能ですが、これはマルチモニタ環境の効率化には寄与しません。むしろ真逆のコンセプトの機能となっています。

PowerToys の FancyZones「ウィンドウ管理」機能は、ウィンドウの移動操作を極限まで効率化してくれますが、「そもそも移動しなくてもいい」という水準の機能は提供していません。

アプリのウィンドウの配置を記憶して自動的に配置する機能を備えたアプリは多数ありますが、これらはモニタの決まった位置にウィンドウを配置することはできても、異なるモニタに配置する機能は提供していません。
